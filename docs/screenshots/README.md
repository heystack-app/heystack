# Screenshots

Drop PNGs here and they will show up in the main README. Capture them with the
app running (`npm run dev`).

**Important:** use neutral demo content, not your personal files, since these go
into a public repo. The included `sample-files` / sample notes are good for this,
or spin up the demo seed data.

Suggested shots (referenced by the README):

| File | What to capture |
|------|-----------------|
| `chat.png` | A question with a streamed answer and the clickable `[n]` citations + Sources list |
| `source-pdf.png` | The source viewer open on a PDF (embedded document + "Cited passage") |
| `source-table.png` | The source viewer showing an Excel sheet rendered as a table, or a markdown preview |
| `scan.png` | The "Scan my computer" modal mid-scan (progress, counts, current file) |
| `dark-light.png` | Side by side or two shots showing the light and dark themes |

Tips:
- Use a clean browser window (no extensions bar), ~1280px wide.
- Capture both light and dark (toggle top-right) where it makes sense.
- Keep them under ~500 KB each (PNG, or compress) so the repo stays light.
